# Mac Qualification — Swift Only (No Bash)

**Patents:** USPTO 19/460,960 | USPTO 19/096,071 — © 2026 Richard Gillespie

---

## Hard Rule

All Mac qualification workflows are **Swift executables only**. No bash, no zsh, no `.sh` qualification scripts.

**Cursor rule:** `GAIAOS/.cursor/rules/mac-qualification-no-bash.mdc`

---

## Architecture

```
GAIAOS/macos/
├── GaiaFusion/          ← MacFusion app (Swift)
├── MacHealth/           ← MacHealth app (Swift)
├── TestRobot/           ← PQ only — Metal GPU tests (Swift)
└── QualificationRunner/ ← Master IQ/OQ/PQ orchestrator (Swift)
```

## QualificationRunner

**Single Swift executable** that runs all qualification phases for both apps:

### IQ (Installation Qualification)
- Check `Package.swift` exists
- Check `MetalRenderer/lib/` (staticlibs) exists
- Check `MetalRenderer/include/` (headers) exists
- Build app via `Process()` running `swift build`
- Verify executable exists (`.build/debug/GaiaFusion` or `.build/debug/MacHealth`)
- Write `evidence/iq/*_iq_receipt.json`

### OQ (Operational Qualification)
- Run `swift test --filter <suite>` for each test suite via `Process()`
- MacFusion: 6 test suites (CellStateTests, SwapLifecycleTests, PlantKindsCatalogTests, etc.)
- MacHealth: 5 tests (FFI lifecycle, epistemic, frame count, null safety, range clamping)
- Write `evidence/oq/*_oq_receipt.json`

### PQ (Performance Qualification)
- **Metal GPU offscreen render test**
- `MTLCreateSystemDefaultDevice()` → verify GPU present
- Create 64×64 BGRA8 texture
- Render clear pass (Tokamak red for MacFusion, Health blue for MacHealth)
- Synchronize managed texture (`blit.synchronize`)
- Read back pixels, verify non-zero
- Write `evidence/pq/*_pq_receipt.json`

### Master Receipt
- Consolidates all IQ/OQ/PQ statuses for both apps
- Writes `evidence/QUALIFICATION_MASTER_RECEIPT.json`

## Why Swift Only?

1. **Consistency:** One language for all Mac operations
2. **Type safety:** Compile-time error checking
3. **Maintainability:** Refactorable, searchable, IDE-supported
4. **Metal integration:** PQ requires Metal API → already Swift → keep everything Swift
5. **No drift:** Prevents bash script proliferation

---

## Exception

**Deploy scripts on remote mesh cells (Hetzner/Netcup)** may use bash:
- SSH remote execution
- No Swift toolchain on mesh nodes
- Deploy ≠ qualification

**Build infrastructure scripts** (e.g. `build_rust.sh`, `build_shaders.sh`) are allowed:
- Build process, not qualification
- May be called from CI/CD pipelines

**On Mac qualification:** Swift only. No exceptions.
