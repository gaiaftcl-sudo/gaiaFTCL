# C4: Full Release Semantics (Grounded)

A **Full Release** is not a single Pass/Fail bit. It is the **concurrence of two independent closure dimensions**. S4 reports (tests, DOCX, Playwright) must not claim planetary closure when C4 only supports identity closure.

## 1. Human / Wallet Axis (Birth Projection)

**Mechanism:** Owl bot, `/moor`, ArangoDB, NATS.

**Closure definition:** A valid wallet signature is registered, `MOOR_COMPLETE` (or configured moor-complete subject) is published, and the DMG gate (`/getmaccellfusion`) is allowed for that Discord identity.

**What the battery proves:** Playwright and onboarding paths exercise the **social handshake** and automation gates. This is **identity / release artifact** closure, not planetary closure.

## 2. Planetary Axis (Living Mesh)

**Mechanism:** Earth ingestor, NATS JetStream, `cell_base` Earth-feed subscriptions.

**Closure definition:** `healthy === total` for the configured Earth pattern set, within the stale window (`DISCORD_EARTH_STALE_SEC`). The `/cell` response reports this as `earth_mooring=(status) (healthy/total patterns)`.

**What the battery does not automatically seal:** If feeds are stale or the ingestor is down, the crystal reports **UNMOORED** or **DRIFTING** regardless of how many users have onboarded. There is no simulated green for the planet layer.

## Truth Markers (Checklist)

- **Social vs Planetary:** A green closure battery confirms the birth projection (identity, tests, DOCX pipeline) has run; it does **not** by itself prove **Earth 11/11** on every crystal.
- **11/11 integrity:** Non-regressing: stale Earth-feed timestamps force **UNMOORED/DRIFTING** in the bot’s own string—honest projection.
- **Domain convergence:** Mooring and fleet state propagate via **NATS and KV**, not by ad-hoc cross-database shortcuts; downstream surfaces observe **ripples** (e.g. fleet digest), not direct Owl table reads in Forge code.
