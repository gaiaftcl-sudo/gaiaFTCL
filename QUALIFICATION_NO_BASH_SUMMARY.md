## ✅ **STATE: Updated — All Bash Scripts Removed**

All Mac qualification workflows are now **Swift executables only**. No bash, no zsh, no `.sh` scripts.

```

└── QualificationRunner/ ← IQ/OQ/PQ master orchestrator (Swift)
```

---

## QualificationRunner (Swift Executable)

**Location:** `GAIAOS/macos/QualificationRunner/`

**What it does:**
1. **IQ (Installation):** Check staticlibs, headers, `Package.swift`, build app, verify executable
2. **OQ (Operational):** Run `swift test --filter` for each test suite via `Process()`
3. **PQ (Performance):** Metal GPU offscreen render tests (64×64 texture, verify non-zero pixels)

**Runs for both apps:** MacFusion + MacHealth

**Build:**
```zsh
cd GAIAOS/macos/QualificationRunner
swift build
```

**Run:**
```zsh
.build/debug/QualificationRunner
```

---

## Why No Bash?

1. **Consistency:** One language (Swift) for all Mac operations
2. **Type safety:** Bash has no types; Swift catches errors at compile time
3. **Maintainability:** Swift code is refactorable; bash scripts drift
4. **Metal integration:** PQ requires Metal API → already Swift → keep everything Swift
5. **No drift:** Prevents bash script proliferation

---

## Exception

**Deploy scripts on remote mesh cells (Hetzner/Netcup)** may use bash because:
- SSH remote execution
- No Swift toolchain guaranteed on mesh nodes
- Deploy is not qualification

**On Mac:** Swift only. No exceptions.
