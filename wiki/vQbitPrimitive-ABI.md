# vQbitPrimitive ABI

The `vQbitPrimitive` struct is the data contract between the USD parser, the Metal renderer, and the Swift layer. Its layout is fixed by `#[repr(C)]` and locked by regression tests on every build.

**Patents: USPTO 19/460,960 | USPTO 19/096,071 — © 2026 Richard Gillespie**

## Struct Definition

```rust
#[repr(C)]

    pub vqbit_truth: f32,          // offset 68, size 4 bytes  ← custom_vQbit:truth_threshold
    pub prim_id: u32,              // offset 72, size 4 bytes
}                                  // total: 76 bytes
```

## USDA Source Format

The parser reads `.usda` files written in two formats. Both are valid.

**Multi-line format:**
```
def Scope "Tokamak" {
    float custom_vQbit:entropy_delta = 0.85
    float custom_vQbit:truth_threshold = 0.92
}
```

**Compact one-liner format:**
```
def Scope "Tokamak" { float custom_vQbit:entropy_delta = 0.85; float custom_vQbit:truth_threshold = 0.92; }
```

The parser handles both formats and any amount of whitespace around the `=` sign. Attribute order within a scope does not matter. A malformed float value (not parseable as `f32`) silently defaults to 0.0 — it never panics (verified by TN-001).

## Colour Mapping

When `upload_geometry_from_primitives()` converts a `vQbitPrimitive` to a `GaiaVertex`, the colour is derived as follows:

```rust
let color = [
    prim.vqbit_entropy.clamp(0.0, 1.0),  // R — entropy_delta
    prim.vqbit_truth.clamp(0.0, 1.0),    // G — truth_threshold
    0.5,                                  // B — fixed
    1.0,                                  // A — opaque
];
```

Values outside [0.0, 1.0] are clamped for display. The raw telemetry values are always preserved in the `vQbitPrimitive` and logged; only the rendered colour is clamped.

## GaiaVertex ABI

The vertex struct passed to the Metal vertex buffer.

```rust
#[repr(C)]
pub struct GaiaVertex {
    pub position: [f32; 3],  // offset 0,  size 12 bytes
    pub color:    [f32; 4],  // offset 12, size 16 bytes
}                            // total: 28 bytes
```

| Field | Offset | Size | Metal attribute |
| --- | --- | --- | --- |
| `position` | 0 | 12 B | `attribute(0)` — `Float3` |
| `color` | 12 | 16 B | `attribute(1)` — `Float4` |
| **Total** | | **28 B** | stride = 28 |

The `MTLVertexDescriptor` in `renderer::new()` is configured to match this layout exactly. The stride is locked at 28 bytes by regression test RG-001.

## Regression Guards

These tests run on every build via `cargo test --workspace`. A failure blocks OQ and the entire qualification chain.

| Test | Assertion | Failure implication |
| --- | --- | --- |
| RG-001 | `size_of::<GaiaVertex>() == 28` | Metal vertex descriptor stride must be updated |
| RG-002 | `size_of::<Uniforms>() == 64` | Metal buffer(1) binding must be updated |
| RG-003 | `size_of::<vQbitPrimitive>() == 76` | Swift FFI boundary broken; C header must be regenerated |
| RG-004 | Field offsets: transform=0, entropy=64, truth=68, prim_id=72 | Swift layer reads wrong memory; full re-qualification required |
| IQ-003 | `size_of::<vQbitPrimitive>() == 76` (parser crate) | Parser and renderer crates disagree on ABI layout |

## Swift FFI

The renderer exposes a C-callable interface for the GaiaFusion Swift layer via `src/lib.rs` compiled as `staticlib`.

```c
/* gaia_metal_renderer.h — auto-generated by cbindgen */
void *gaia_metal_renderer_create(void);
void  gaia_metal_renderer_destroy(void *ptr);
void  gaia_metal_renderer_render_frame(void *ptr);
void  gaia_metal_renderer_set_tau(void *ptr, uint64_t block_height);
```

`set_tau` sets τ — the sovereign time axis expressed as Bitcoin block height. τ is updated by GaiaFusion Swift on every `gaiaftcl.bitcoin.heartbeat` NATS message (~every 10 minutes). τ = 0 means genesis / not yet synchronised.

Any change to `vQbitPrimitive` that alters its size or field offsets requires:
1. Updating `gaia_metal_renderer.h` via `cbindgen`
2. Re-running the full PQ test suite
3. Re-signing all GxP evidence

## Configuration Management

The `vQbitPrimitive` ABI is controlled item CI-007. Changes require a Change Control Record (CCR) and trigger full PQ re-execution. No change to field names, types, sizes, or offsets may be made without a CCR and PQ sign-off.
